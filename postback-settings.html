Postback Settings

Optional URL Parameters

The URL can include the value of various parameters. Simply include the placeholder for the parameters in the URL. Prior to calling the URL, the placeholders for the parameters will be replaced with their values

For example if you wanted to populate the program ID in the URL you would set it up as follows: https://www.domain.com?publisherId={publisherId}. When the URL is called, {publisherId} would be populated with the value provided

Example Implementation
// Your postback URL with parameters
https://yourdomain.com/tracking?pid={publisherId}&tid={transactionId}&amt={amount}&checksum={checksum}

Supported Custom Fields

Parameter Description
{transactionId} Order's unique identifier
{publisherId} Publisher company ID
{merchantId} Merchant ID in FatCoupon Partner system
{orderNo} Order number
{amount} Order amount value, in USD
{commission} Order commission value, in USD
{clickId} Click ID from redirect link
{status} Order status: Pending, Payable, Canceled, or Cashed
{orderDate} Order date time in UTC, format: YYYY-MM-DDTHH:mm:ss.sssZ
{subId1} to {subId5} Tracking parameters appended in tracking URL
Checksum Security Parameter Recommended

The {checksum} parameter provides an additional layer of security by generating an MD5 hash of your transaction data combined with a secret key

// Checksum formula
MD5(secretKey + '&' + transactionId + '&' + orderNo + '&' + amount + '&' + commission + '&' + secretKey)

This ensures that postback requests are authentic and haven't been tampered with during transmission

Sample Postback URL
// Complete example with all parameters
https://yourdomain.com/tracking?
  pubId={publisherId}
  transId={transactionId}
  order={orderNo}
  sale={amount}
  comm={commission}
  click={clickId}
  status={status}
  date={orderDate}
  checksum={checksum}
